Import / Export Helper

MongoDB provides two mechanisms for importing and exporting data. One way is via the mongoimport and mongoexport utilities. These allow you to import and export JSON and CSV representations of your data. The other way is with mongorestore and mongodump utilities which deal with binary dumps.

In this tab we provide pre-filled strings for the commands that we find most useful.

Copy and paste from below to import or export from this database. For a full list of options that can be used with these commands, please see MongoDB's documentation on this subject.

Binary

Import collection

mongorestore -h ds053438.mongolab.com:53438 -d challenge_u_tester -c users -u <user> -p <password> users.bson

Export collection

mongodump -h ds053438.mongolab.com:53438 -d challenge_u_tester -c users -u <user> -p <password> -o <output directory>

JSON

Import collection

mongoimport -h ds053438.mongolab.com:53438 -d challenge_u_tester -c users -u <user> -p <password> --file <input file>

Export collection

mongoexport -h ds053438.mongolab.com:53438 -d challenge_u_tester -c users -u <user> -p <password> -o users.json

CSV

Import collection

mongoimport -h ds053438.mongolab.com:53438 -d challenge_u_tester -c users -u <user> -p <password> --file <input .csv file> --type csv --headerline

Export collection

mongoexport -h ds053438.mongolab.com:53438 -d challenge_u_tester -c users -u <user> -p <password> -o users.csv --csv -f <comma-separated list of field names>

Indexesfull docs

            

              From the "Indexes" tab you can manage indexes for this
              collection. Index configuration is stored in the
              system.indexes
              collection for this database, but you should create indexes either via this page or the ensureIndex method.
	    

            
Standard Indexes full docs

            

              From this UI, you can create simple or compound indexes that are optionally unique and/or sparse.  You can even index a key inside an embedded object (e.g. address.city).
            

            

	      When you see indexed fields that look like:
            

            

              { <field-name> : <-1 or 1> }
            

            

              ...the 1 indicates an ascending sort order and -1 a descending sort order. Sort order only matters for compound indexes.
            


            
Geospatial Indexes full docs

            

              You can also create a 2d or 2dsphere geospatial index here. 2dsphere index supports queries that calculate geometries on an earth-like sphere and is only available in MongoDB 2.4 and above.
            

            

              Please note that MongoDB currently allows only one geospatial index per collection.  If you'd like to replace an existing index, delete it and add another one.

Conditional Operators

              

                
$gt  - greater than <value>, $lt - less than <value>,

                
$gte - greater than or equal to <value>,

                
$lte - less than or equal to <value>

                
e.g. { "size" : {"$lt": 3} }

              

              

                
$ne - not equals <value>

                
e.g. { "status" : {"$ne": "complete"} }

              

              

                
$mod - mod <divisor> is <value>

                
syntax: { <field> : {"$mod":[<divisor>, <value>]} }

                
e.g. { "size" : {"$mod": [10, 1]} }

              

              

                
$in - in <array>, $nin - not in <array>

                
e.g. { "size" : {"$in": [1, 2, 3]} }

              

              

                
$exists - field present or missing

                
e.g. { "nicknames" : {"$exists": false} }

              

              

                
$type - BSON typereferenceis <type number>

                
e.g. { "name" : {"$type": 2} }

                
     // matches if "name" is a String

              

              

                
$all - array matches all of the elements in <array>

                
e.g. { "nicknames": { "$all": ["Liz", "Beth"] } }

              

              

                
$size - array has specified # of elements

                
e.g. { "nicknames" : {"$size": 3} }

              

              

                
$elemMatch - array matches criteriafull example

                
e.g. { "f": {"$elemMatch": {"s":"dot","c":"red"}} }

              

              

                
$near - ordered by distance from [ lat, long ]docs

                
e.g. { "loc": {"$near": [ 38, -122 ] } }

Unique

            

              A unique flag of true signifies a unique index. The _id_ index is always unique.
            

            

              Parentheses indicate an implicit value. Documents representing index definitions do not explicitly include unique: false, so the value of unique is "(false)."
            

          

          

	    
            
Sparse

            

              A sparse flag of true signifies a sparse index, which
              only contains documents with the indexed field.  When sorting or
              filtering on a sparsely indexed field, any documents that are
              missing that field will be omitted from the query results.
            

            

              Note that a sparse index can only have one field.
            

            

              Parentheses indicate an implicit value. Documents representing index definitions do not explicitly include sparse: false, so the value of sparse is "(false)."
            

          

          

	    
            
Min / Max

            

              If min and max are blank, the index assumes lattitude/longitude and is thus configured for a -180 to 180 value range.
